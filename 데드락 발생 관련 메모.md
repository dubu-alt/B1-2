$ ps -ef | grep agent-app-leak

&#x20;  vnker    1148     945 cons1    19:11:14 grep agent-app-leak 



vnker: 명령어를 실행한 사용자 계정 아이디

1148: grep 명령어의 프로세스 ID (PID)

945: 이 명령을 실행한 부모 프로세스 ID (PPID)

cons1: 명령어가 실행된 터미널 이름

19:11:14: 명령어가 실행된 시각

grep agent-app-leak: 실행된 명령어 본문



ps -ef는 시스템에서 실행 중인 모든 프로세스를 상세히 보여주는 명령어입니다.

ps: Process Status (프로세스 상태)

\-e: Everything / Every process (모든 프로세스 선택)

\-f: Full-format listing (상세한 형태로 출력)



PID와 PPID의 개념

* PID (Process ID): 프로세스 고유 번호입니다. 주민등록번호처럼 시스템 안에서 각 프로세스를 식별합니다.
* PPID (Parent Process ID): 부모 프로세스의 고유 번호입니다. 나를 태어나게 한 프로세스의 PID를 가리킵니다.



* 자식 프로세스 (grep): PID는 1148입니다. 사용자가 엔터를 치는 순간 시스템이 grep이라는 자식 프로세스를 일시적으로 생성한 것입니다.
* 부모 프로세스 (터미널 쉘): PPID는 945입니다. 이는 1148을 실행시킨 명령줄 창(예: bash, zsh 등)의 고유 번호입니다.

ps -ef | grep 945
